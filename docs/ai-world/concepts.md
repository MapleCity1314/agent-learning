# AI 基础概念入门

> 你每天都在用 AI 工具，但可能从来没想过它背后的逻辑。这篇文章帮你把最核心的几个概念搞明白——不是为了变成技术专家，而是让你用得更好。

---

## 大语言模型：你天天在用的那个东西

LLM，全称 Large Language Model，中文叫「大语言模型」。你用的 Claude、ChatGPT、文心一言，都属于这一类。

### 它到底在做什么

最简单的理解：它读了互联网上海量的文字，从中学会了「什么话该接什么话」。所以你说一句，它能接出一个看起来很合理的回答。

注意，是「看起来很合理」。它并不是真的懂了你说的话，更像是一个语感极好的人在凭直觉写作——大多数时候写得不错，但偶尔会很自信地写出错的东西。这种现象有个专门的名字叫「幻觉」。

还有几个常见误区值得说一下：

- 它**不是搜索引擎**。搜索引擎是去找已有的网页，LLM 是自己生成回答。
- 它**不知道今天发生了什么**。它的知识有截止日期，除非你用的版本支持联网搜索。
- 它**不保证正确**。越是你不熟悉的领域，越要自己核实它说的话。

### 主流的几个模型

说几个你可能会碰到的名字：

- **Claude**（Anthropic 出的）—— 中文写作能力不错，逻辑比较强，你现在用的就是这个
- **ChatGPT / GPT-4o**（OpenAI 出的）—— 最早火起来的，周边工具最多
- **Gemini**（Google 出的）—— 和 Google 的服务绑得比较紧
- **文心一言**（百度）和**通义千问**（阿里）—— 在国内用比较方便，不用翻墙

你不需要纠结哪个最好。它们各有长处，试过才知道哪个适合你的场景。

## 多模态：不只是聊天，还能看图听声音

早期的 AI 只能处理文字。现在的模型越来越多地能处理图片、音频甚至视频，这种能力叫「多模态」。

这对你意味着什么？举个你马上能用到的例子：

你拍了一张照片，直接发给 Claude，说「帮我根据这张图的氛围写一条小红书文案」。它能看见图里有什么、是什么感觉，直接写出来。比你先用文字描述照片、再让它写，效果好太多了。

除了理解图片，现在还有专门生成图片的工具（比如即梦、Midjourney）、生成语音的（ElevenLabs）、生成视频的（可灵、Sora）。这些工具发展很快，但目前最实用的还是「把图片发给 AI 让它理解」这个方向。

## 用 AI 的三种方式

这部分最值得花时间理解。同样是 AI，用法不同，能做的事情差距非常大。

### Chat：一问一答

你现在最熟悉的方式。打开 Claude.ai，说一句，它回一句，像发微信。

**好处是简单直接。** 想到什么问什么，改文案、头脑风暴、问问题，随时能用。

**但它有几个天然的限制：**
- 每次开新对话就像跟一个陌生人重新聊，之前说过的它全忘了
- 做完了你得自己复制粘贴结果
- 只能一步一步来，做不了复杂的连续操作

你现在做的就是这种：打开 Claude → 聊 → 复制文案 → 粘贴到小红书。它够用，但不是唯一的方式。

### Workflow：把固定步骤串起来

如果你发现自己每天都在重复同样几步操作，就可以考虑 Workflow 了。

打个比方：你在奶茶店点了杯奶茶，从下单到拿杯，中间有一套固定流程——接单、备料、制作、出杯——每步自动触发下一步。Workflow 做的是类似的事情，只不过每一步是 AI 在执行。

比如你可以设一条流程：输入一个选题 → 自动生成 5 个标题 → 自动写正文 → 自动打标签。设好之后，每次只要丢进去一个选题就行了。

**适合什么场景？** 固定的、重复的任务。如果一件事你做了三次以上、每次步骤都差不多，就值得用 Workflow 自动化。

想试试的话，可以看看 [Coze](https://www.coze.cn)（国内能用）或 [Dify](https://dify.ai)。

### Agent：给个目标就行

这是目前最强大的形式，但也最不成熟。

Chat 需要你一步一步告诉它做什么。Agent 不一样——你只需要说一个目标，它自己想办法。

比如你说「帮我整理下周的内容计划」，一个 Agent 可能会：自己去看你过去内容的数据 → 分析哪类表现好 → 搜索最近的热门话题 → 综合写出一份计划。全程你不需要管中间每一步怎么做。

**说实话，目前大多数人还用不太到 Agent。** 它的概念很诱人，但现在的工具还没完全成熟到「说句话就全自动」的程度。不过方向是确定的，了解这个概念会帮你在新工具出来时更快判断它在做什么。

你现在能接触到的、带点 Agent 味道的东西：
- Claude Projects —— 能记住你们之前的对话上下文
- Claude Code —— 你告诉它做一个网站，它自己写代码、运行、改 bug，直到搞定

## 提示词：同样的工具，说法不同结果差很多

这是你现在马上就能用上的技巧。

同一个任务，你怎么说，直接决定 AI 回答的质量。看个对比：

**随便说：**
```
帮我写个文案
```

**认真说：**
```
帮我写一条小红书文案，主题是「宿舍窗边的下午，阳光打在书桌上」，
风格温暖治愈，目标读者是大学生，
需要：3 个标题备选 + 正文 150 字 + 5 个相关标签
```

差距在哪？第二个给了四样东西：**主题**（写什么）、**风格**（什么感觉）、**读者**（给谁看）、**格式**（要什么形式的结果）。AI 知道该往哪个方向走了。

还有几个特别好用的小技巧：

**给它看例子，比描述管用。** 与其说「标题要有悬念感」，不如直接贴一个你觉得好的标题让它参考：
```
帮我写标题，参考这个风格：
「为什么我拍的照片总差一点？原来差在这里」
主题是：手机修图入门
```

**不满意直接说哪里不好。** 别说「重新写一个」，要说「这个标题太平了，重新写 3 个，要更有悬念感」。越具体，AI 越知道怎么改。

**复杂任务拆开来做。** 别一口气要求标题+正文+标签全写完。先让它给 5 个标题备选，你挑一个，再让它写正文。分步走质量更高。

## 上下文：让 AI 记住你是谁

这个概念不复杂，但很多人忽略它。

你跟 AI 聊天的时候，同一个对话里前面说过的话它都记得。但一旦你开了个新对话，之前说的全没了——对它来说你是个全新的陌生人。

这就导致一个烦人的问题：每次开新对话你都得重新介绍自己的账号方向、风格偏好、目标读者这些背景信息。

**怎么解决？用 Claude Projects。**

在 Claude.ai 左侧创建一个 Project，在设置里写上你的固定背景（比如「我运营一个生活/摄影方向的小红书账号，目标读者是大学生，内容风格温暖治愈」）。之后在这个 Project 里开的所有对话，Claude 都会自动带上这些信息。

相当于你有了一个「已经了解你的 Claude」，不用每次重新介绍。这个小设置能省很多时间，而且回答质量会明显提高。

## 还有几个概念，了解就好

下面这两个概念你可能会在各种文章里看到，但对你现在的阶段来说，知道是什么就行了，不需要深入研究。

### 工具调用

AI 本身只能处理文字。但它可以在对话中「调用」外部工具——就像你的手机装了高德地图之后才有导航功能一样。

你现在已经在用了：Claude 的联网搜索功能就是一种工具调用。你问「最近小红书上什么话题热门」，它会自己去搜，而不是靠可能过时的训练数据来猜。

### MCP（连接器）

全称 Model Context Protocol。你可以理解为 AI 和其他软件之间的标准接口。

以前每个软件想接入 AI，都得各自开发一套对接方式。MCP 相当于制定了一套通用标准——做一次适配，所有支持 MCP 的 AI 就都能用。

这意味着将来你可能可以直接对 AI 说：「把我小红书数据最好的 3 条内容整理成报告，存到 Notion 里」——AI 能自己完成所有步骤。这个生态还在建设中，目前 Notion、Google Drive 等工具已经支持。

## 这些概念怎么串起来

用一个你可能会遇到的场景来理解它们的关系：

**你想每周自动生成下周的内容计划。**

首先你需要一个 **LLM** 来理解你的需求、生成内容——这是核心引擎。如果你把上周发布的图片直接发给它看，用到的就是**多模态**能力。

你现在手动做这件事，每次都在 Chat 里说一遍需求，这是 **Chat** 模式。如果你把这几步固定下来自动执行，就变成了 **Workflow**。如果你连每一步怎么做都不想管、只想说个目标就出结果，那是 **Agent** 的方向。

你给 AI 说需求的方式，是**提示词**。你提前在 Project 里写好的账号背景，是**上下文**。AI 去搜热门话题、读你的数据表，用的是**工具调用**。这些工具之间用统一标准对接，靠的是 **MCP**。

> 不需要一次性记住所有概念。先把 LLM、Chat、提示词这三个搞明白，其他的用到了再回来看。
